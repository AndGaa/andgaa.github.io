Timetrack Rapport

A Power BI rapport for monitoriring time spend on projects and tasks, integrating data from Clickup via API.

  • Role: Data Analyst
  • Stack: PowerBi, M, DAX
  • Skills: Data Modelling, Data Visualization, API interaction

Project Overview

Overview

The project aimed to solve a key business need for managing how time was spend on projects and tasks. By creating a Power BI rapport that integrated with Clickup's API and E-conomic API, we provided stakeholders with a much needed overview of time allocation and resource utilization.

Key Features

Key Features

  • Role-based access control
  • On Demand refresh through Power Automate Flows
  • Self Service Data Visualization

Data Model

The Model

The stakeholders decided to use Clickup as their primary project management tool, which presented a challenge in extracting and transforming the data into a format suitable for Power BI. All the project meta data was kept in Clickup in a pseudo structure, to keep the structure of Clickup separate from the project structure needed for reporting. I developed a custom data pipeline using Power Query M language to interact with Clickup's API, to extract project data and time tracking data, and establish a relationship through a bridge table. This allowed for accurate aggregation and analysis of time spent on projects as well as individual employee contributions.

Challenges & Solutions

One of the main challenges of the project was dealing with the limitations of Clickup's API, which had restrictions on the amount of data that could be retrieved in a single request. To overcome this, I implemented a pagination mechanism in the data extraction process, allowing for the retrieval of large datasets in manageable chunks. The API could return denormalized timetracking data, which allowed me to stage data, before transforming it into a more analysis-friendly format in Power BI. This made the tedious task of retriveving tasks data by the 100s redundant, and allowed for a more efficient data refresh process.
